More about Binary Search:

  • Binary Search is a highly efficient algorithm for searching a key element in a sorted array. It works by repeatedly dividing the array into two halves and checking whether the target element lies in the left or right half, drastically reducing the search space.
  • Key Features:
    1. Sorted Array Requirement: The array must be sorted for the binary search algorithm to work.
    2. Divide-and-Conquer Strategy: The search range is halved at every step, improving efficiency.
  • Time Complexity:
    • Best Case: O(1) (target found at the middle in the first step).
    • Worst Case: O(logn) (dividing the search space until one element remains).
  • Space Complexity:
    • Iterative Approach: O(1).
    • Recursive Approach: O(log n) (due to stack calls).
  • Algorithm Steps:
    1. Set two pointers: low (starting index) and high (last index of the array).
    2. Compute the middle index: Compare the target element with the middle element:
    3. If equal: Return the middle index (element found).
    4. If target is smaller: Focus on the left half (high = middle - 1).
    5. If target is larger: Focus on the right half (low = middle + 1).
    6. Repeat the process until: The target is found, or The low pointer exceeds high, meaning the target is not in the array.
